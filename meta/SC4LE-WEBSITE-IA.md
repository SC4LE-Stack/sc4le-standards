## 1. Purpose of This Document

This Website Information Architecture (IA) defines the structure, hierarchy, and content model for the SC4LE website.  
It ensures that all public-facing content is:

- aligned to the SC4LE Standard  
- consistent with the SC4LE brand system  
- governed under the SC4LE meta-governance layer  
- easy to navigate, minimal, and strategically structured  

This IA is part of the SC4LE Standards Repository and must evolve under the same governance rules.

---

## 2. Golden Source Alignment

All website content must align to the SC4LE Standard stored in the SC4LE-Standards repository:

**https://github.com/SC4LE-Stack/sc4le-standards**

The website is a representation of the standard — not a source of truth.

No new concepts, terminology, services, or operating model elements may be introduced outside the standard.

---

## 3. Top-Level Navigation Structure

The SC4LE website must use the following top-level structure:

1. **Home**
2. **How SC4LE Works**
3. **Programmes**
4. **Services**
5. **Resources**
6. **About**

Each top-level section is defined below.

## 5. Page Structure Rules

All SC4LE pages must follow this structural pattern:

1. **Hero Section**
   - Minimal
   - Strategic
   - Outcome-focused

2. **Context / Problem**
   - Cognitive load
   - Decision latency
   - Governance fragmentation
   - Scaling challenges

3. **SC4LE Response**
   - Shared principles
   - Federated governance
   - AI-enabled sensing
   - Continuous improvement

4. **Model Component / Service Detail**
   - Clear definitions
   - No invented terminology

5. **Benefits**
   - Faster flow
   - Reduced rework
   - Clearer decision pathways
   - Stronger regulatory alignment
   - Sustainable capability uplift

6. **CTA**
   - Enable Clarity. Accelerate Flow.

## 6. Tone, Style, and Visual Rules

All pages must follow the SC4LE brand system:

### Tone
- Clear  
- Strategic  
- Modern  
- Confident  
- Enabling  
- Outcome-focused  

### Visual
- Use official SC4LE spiral logo  
- Use charcoal, teal, electric blue  
- Use minimal, geometric layouts  
- Use spiral-inspired arcs for flows  

### Language
- Use SC4LE terminology exactly as defined in the standard  
- No invented concepts  
- No generic agile jargon
