# SC4LE Website Content Matrix  
### Information Architecture, Page Types, Content Blocks & Governance  
© 2026 SC4LE Limited

---

## 1. Purpose of This Document

This document defines the **information architecture (IA)**, **page types**, **content blocks**, **metadata requirements**, and **governance rules** for the SC4LE website.

It ensures the website remains:

- coherent  
- principle‑aligned  
- brand‑consistent  
- governed  
- scalable  
- aligned to the SC4LE Standard  

This file is the **golden source** for all website structure and content.

---

## 2. Website Information Architecture (IA)

The SC4LE website is structured around four primary IA layers:

1. **Homepage**  
2. **Who We Help** (navigation category)  
   - Startups  
   - Scale-ups  
   - SMEs  
   - Enterprise  
3. **Services**  
4. **About / Contact / Legal**  

Each page type is governed by a defined template and content model.

---

## 6. Content Blocks by Page Type

Each page type uses a defined set of content blocks.

---

### 6.1 Homepage

**Required Blocks**
- Hero  
- SC4LE Value Proposition  
- Who We Help Overview  
- Services Overview  
- Operating Model Snapshot  
- Proof Points  
- CTA  

**Optional Blocks**
- Featured Insights  
- Featured Case Studies  

---

### 6.2 Programme Page (Startups / Scale-ups / SMEs / Enterprise)

**Required Blocks**
- Hero  
- Segment Pain Points  
- SC4LE Programme Overview  
- Value Proposition (segment‑specific)  
- Operating Model Alignment  
- Programme Structure  
- Outcomes & Benefits  
- Proof Points  
- CTA  

**Optional Blocks**
- Case Studies  
- Insights for this Segment  

**Metadata Requirements**
- target_market: startups | scale-ups | smes | enterprise  
- page_type: programme  
- canonical_diagrams: []  
- value_props: []  

---

## 7. Metadata Requirements (Global)

All pages must include:

- title  
- slug  
- description  
- page_type  
- owner  
- updated  
- canonical_url  
- tags  
- schema_type  

Programme Pages must also include:

- target_market  
- value_props  
- canonical_diagrams  

---

## 8. Governance Rules

### 8.1 Minor Change
- copy edits  
- formatting  
- metadata updates  
**Approval:** Editor

### 8.2 Moderate Change
- new sections  
- updated diagrams  
- new value propositions  
**Approval:** Editor + SSB simple majority

### 8.3 Major Change
- new page types  
- new IA structure  
- new navigation  
- new programme structure  
**Approval:** SSB unanimous approval  

---

## 9. AI Usage Rules

AI may support content creation but must not:

- invent new SC4LE concepts  
- alter the operating model  
- create new page types  
- generate new value propositions  
- contradict the SC4LE Standard  
- generate alternative brand assets  

AI outputs must be reviewed by Editors.

---

## 10. Alignment Requirements

All pages must align to:

- SC4LE Shared Principles  
- SC4LE Brand Governance  
- SC4LE Messaging Governance  
- SC4LE Diagram Governance  
- SC4LE Content Governance  

Programme Pages must explicitly reference:

- segment‑specific value propositions  
- operating model alignment  
- SC4LE principles
