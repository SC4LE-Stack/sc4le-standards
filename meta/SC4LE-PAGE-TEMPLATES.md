## 1. Purpose of This Document

This document defines the canonical page templates for the SC4LE website.  
These templates ensure that all SC4LE pages:

- follow a consistent structure  
- align to the SC4LE Standard  
- use approved terminology  
- reflect the SC4LE brand system  
- maintain strategic clarity and minimalism  

This file is part of the SC4LE meta-governance layer and must evolve under the same governance rules as the SC4LE Standard.

---

## 2. Golden Source Alignment

All page templates must align to the SC4LE Standard stored in the SC4LE-Standards repository:

**https://github.com/SC4LE-Stack/sc4le-standards**

No template may introduce new SC4LE concepts, terminology, or structures outside the standard.

---

## 3. Universal Page Structure Rules

All SC4LE pages must follow this structural pattern:

1. **Hero Section**  
   - Minimal  
   - Strategic  
   - Outcome-focused  
   - One headline + one subheadline + CTA  

2. **Context / Problem**  
   - Cognitive load  
   - Decision latency  
   - Governance fragmentation  
   - Scaling challenges  

3. **SC4LE Response**  
   - Shared principles  
   - Federated governance  
   - AI-enabled sensing  
   - Continuous improvement  

4. **Model Component / Service Detail**  
   - Clear definitions  
   - No invented terminology  
   - Aligned to the SC4LE Standard  

5. **Benefits**  
   - Faster flow  
   - Reduced rework  
   - Clearer decision pathways  
   - Stronger regulatory alignment  
   - Sustainable capability uplift  

6. **CTA**  
   - Enable Clarity. Accelerate Flow.

These rules apply to all templates below.
