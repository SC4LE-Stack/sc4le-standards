## 1. Purpose of This Playbook

This playbook defines how SC4LE website content is created, governed, updated, and published.  
It ensures that all public-facing SC4LE material remains aligned to the SC4LE Standard, which is maintained in the SC4LE-Standards GitHub repository.

This document is part of the SC4LE meta-governance layer and is governed under the same rules as the SC4LE Standard.

---

## 2. Golden Source Rule (Non-Negotiable)

The SC4LE-Standards GitHub repository is the **single source of truth** for:

- SC4LE Operating Model  
- SC4LE Governance Model  
- SC4LE Shared Principles  
- SC4LE Terminology  
- SC4LE Diagrams  
- SC4LE Definitions and Schemas  
- SC4LE Value Propositions  
- SC4LE Messaging Standards  

Authoritative Repository:  
**https://github.com/SC4LE-Stack/sc4le-standards**

All website content must align to the repository.  
No new SC4LE concepts, terminology, services, or structures may be invented outside the standard.

---

## 3. Relationship Between the Standard and the Website

The SC4LE Standard defines:

- what SC4LE *is*  
- how SC4LE *works*  
- how SC4LE *governs*  
- how SC4LE *evolves*  

The website is a **representation** of the standard, not a source of truth.

The website must always reflect the latest version of the SC4LE Standard.

---

## 4. Publishing Workflow

### Step 1 — Draft Content
Create raw content in any format (notes, bullets, rough text).

### Step 2 — Apply the SC4LE Master Publishing Prompt

To avoid breaking this Markdown file, the prompt is shown with escaped backticks:

\`\`\`
Use the SC4LE Workspace Knowledge as the single source of truth.
The sc4le-standards GitHub repository is the authoritative golden source for all SC4LE content.
Do not introduce new concepts, terminology, or services beyond what the repository defines.
Transform the content below into a polished, client-ready webpage.
Maintain SC4LE’s tone, structure, and strategic enablement principles.
Make the page clean, modern, and web-ready.
\`\`\`

### Step 3 — Validate Against the Standard
Check alignment with:

- Operating Model  
- Governance Model  
- Shared Principles  
- Terminology  
- Diagrams  
- Value Propositions  

### Step 4 — Publish

### Step 5 — Version Tracking
When the SC4LE Standard updates:

1. Update the GitHub repo  
2. Tag a new version  
3. Regenerate affected website pages  
4. Publish updated content

- Matches the SC4LE Operating Model
